GitHub发布操作详解

GitHub操作四步骤

1 创建仓库（Repository）

仓库就像是一个文件夹，用来存放我们的网页文件

示例： github.com/OtokoNoIzumi/ai_learning

2 上传文件到仓库

把我们用Cursor制作的网页文件上传到GitHub仓库

两种方式： 网页版上传（入门推荐）或 Git客户端（进阶）
参考资料： Git官方中文教程 1.6、2.1、2.2章节

3 开启GitHub Pages功能

让GitHub把我们的代码变成可以访问的网站

操作路径： Settings → Pages → Build and deployment → Branch → Save
等待时间： 几分钟后刷新即可访问

4 理解发布网站的网址结构
otokonoizumi.github.io/ai_learning/report
otokonoizumi

GitHub用户名

ai_learning

仓库名称

report

文件路径

小技巧： 如果上传的文件名是 index.html，则不需要在网址后面加文件名；对于html文件，也可以省略.html后缀

Git客户端命令行操作进阶

熟悉网页版操作后，可以尝试使用Git命令行工具，提升操作效率。命令行操作让你更好地理解Git的工作原理。

基础操作流程

1
打开Git命令行

选择一个文件夹，右键菜单选择"Git Bash Here"

2
克隆仓库到本地
git clone https://github.com/OtokoNoIzumi/ai_learning.git

替换成你自己创建的仓库地址

3
添加文件到暂存区
git add .

添加所有文件，或者：

git add report.html

只添加指定文件

4
提交改动
git commit -m "改动说明文字"

用简短文字描述你的修改内容

5
推送到远程仓库
git push

将本地改动同步到GitHub

实际项目创作 - SVG二维码分享卡片

项目设计

技术选择

• SVG格式：矢量图形，可缩放，文件小

• Web工具：无需Python环境，基于浏览器

• AI辅助：智能生成和优化设计

入门应用场景

• 个人名片设计

• 宣传海报

技术路径

SVG编辑的优势

  • • 可视化编辑：类似设计软件，但用代码描述
  • • AI友好：文本格式，容易被AI理解和修改
  • • 跨平台兼容：任何现代浏览器都可以显示
  • • 专业输出：可直接用于印刷或数字展示

无Python解决方案

1. 二维码生成（推荐SVG格式输出工具）

  • • QR Planet：免费专业SVG输出
  • • Aquiestoy：免费支持自定义样式
  • • 条妈妈：国内工具，SVG批量生成

2. SVG编辑和预览

  • • AI辅助生成基础框架（Cursor）
  • • 浏览器内直接预览（.html文件包含SVG）
  • • 在线预览工具：onlinestool.com

实操流程

1
提出制作需求

向AI描述要制作的内容类型和基本需求，由AI给出基本内容框架

2
调整内容细节

类似编辑网页的方式，对生成的内容进行精细化调整和优化

3
整合非文字内容

添加二维码、图片等非纯文字元素，完善整体设计

4
保存输出

最终检查后保存文件，准备用于实际应用
